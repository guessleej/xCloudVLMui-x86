### 2.2 Backend 健康端點實作

```python
@app.get("/api/health")
async def health_check(db: AsyncSession = Depends(get_db)) -> dict:
    checks = {}
    
    # DB 連線檢查
    try:
        await db.execute(text("SELECT 1"))
        checks["db"] = "ok"
    except Exception as e:
        checks["db"] = f"error: {e}"
    
    # llama.cpp 連線檢查
    try:
        async with httpx.AsyncClient(timeout=5) as client:
            resp = await client.get(f"{settings.LLAMA_CPP_HOST}/health")
            checks["llm"] = "ok" if resp.status_code == 200 else "degraded"
    except Exception:
        checks["llm"] = "unreachable"
    
    # MQTT Broker 連線檢查
    checks["mqtt"] = "ok" if mqtt_service.is_connected() else "disconnected"
    
    overall = "healthy" if all(v == "ok" for v in checks.values()) else "degraded"
    
    return {
        "status": overall,
        "timestamp": datetime.utcnow().isoformat(),
        "version": settings.APP_VERSION,
        **checks
    }
```

### 3.2 事件來源分類

| 來源（source） | 說明 | 觸發條件 |
|---------------|------|----------|
| `middleware` | HTTP 請求自動記錄 | 每次 API 呼叫 |
| `mqtt` | MQTT 訊息接收/告警 | 感測器資料到達 |
| `pipeline` | 推論流水線執行 | 觸發 VLM 檢測 |
| `system` | 服務啟停事件 | 容器啟動/關閉 |
| `manual` | 人工標記事件 | 管理員操作 |

### 3.4 Event Center UI

Event Center 頁面（`/events`）提供以下功能：

- **時間軸過濾**：選擇時間範圍查詢

### 4.2 syslog duration_ms 監控查詢

```sql
-- 查詢最慢的 API（過去 24 小時）
SELECT 
    path,
    method,
    COUNT(*) as request_count,
    AVG(duration_ms) as avg_ms,
    MAX(duration_ms) as max_ms,
    SUM(CASE WHEN status_code >= 500 THEN 1 ELSE 0 END) as error_count
FROM syslog
WHERE timestamp > datetime('now', '-24 hours')
GROUP BY path, method
ORDER BY avg_ms DESC
LIMIT 20;

-- 查詢錯誤率
SELECT 
    strftime('%H', timestamp) as hour,
    COUNT(*) as total,
    SUM(CASE WHEN status_code >= 500 THEN 1 ELSE 0 END) as errors,
    ROUND(100.0 * SUM(CASE WHEN status_code >= 500 THEN 1 ELSE 0 END) / COUNT(*), 2) as error_rate
FROM syslog
WHERE timestamp > datetime('now', '-24 hours')
GROUP BY hour
ORDER BY hour;
```

### 5.2 感測器資料流監控

```python
# MQTT on_message callback 中的監控邏輯
async def on_message(self, client, userdata, msg):
    start = time.time()
    
    try:
        payload = json.loads(msg.payload)
        device_id, sensor_type = self._parse_topic(msg.topic)
        
        # 寫入 SensorReading
        await self.sensor_repo.create(device_id, sensor_type, payload["value"])
        
        # 閾值檢查
        config = await self.mqtt_config_repo.get_by_topic(msg.topic)
        if config and self._exceeds_threshold(payload["value"], config):
            await self.alert_service.create_mqtt_alert(device_id, sensor_type, payload["value"], config)
        
        # 記錄成功事件
        await write_syslog(
            level="INFO",
            source="mqtt",
            message=f"Received {sensor_type}={payload['value']} from {device_id}",
            duration_ms=(time.time() - start) * 1000
        )
    
    except Exception as e:
        await write_syslog(
            level="ERROR",
            source="mqtt",
            message=f"MQTT message processing error: {e}",
            extra={"topic": msg.topic, "payload": str(msg.payload[:100])}
        )
```

## 6. 告警通知機制

### 6.1 內建 UI 告警

| 告警類型 | 觸發條件 | 顯示位置 | 持久化 |
|----------|----------|----------|--------|
| MQTT 閾值超限 | 感測器值 > 閾值 | AlertBanner + Drawer | ✅ EquipmentAlert |
| VLM 推論異常 | 推論錯誤或低置信度 | AlertBanner | ✅ EquipmentAlert |
| 服務不健康 | `/api/health` 回傳 degraded | StatusBar | ❌ 記憶體 |
| Pipeline 失敗 | 任一 Stage 回傳 error | Equipment Drawer | ✅ InspectionReport |

### 6.2 告警等級定義

| 等級 | 定義 | UI 顏色 | 自動解除 |

| `info` | 資訊性通知 | 藍色 | 24h |

## 8. 日誌保留策略

### 8.1 syslog.db 自動清理

```python
# 排程任務：每日 02:00 執行
async def cleanup_old_syslogs():
    """保留 90 天內的日誌，刪除舊記錄"""
    cutoff = datetime.utcnow() - timedelta(days=90)
    
    conn = sqlite3.connect(settings.SYSLOG_DB_PATH)
    cursor = conn.execute(
        "DELETE FROM syslog WHERE timestamp < ?",
        (cutoff.isoformat(),)
    )
    deleted = cursor.rowcount
    conn.commit()
    
    await write_syslog(
        level="INFO",
        source="system",
        message=f"Syslog cleanup: deleted {deleted} records older than 90 days"
    )
```

### 8.3 保留策略摘要

| 資料類型 | 保留期限 | 清理機制 |
|----------|----------|----------|
| syslog.db HTTP 記錄 | 90 天 | 每日排程清理 |
| Docker container logs | 最近 5 個 × 100MB | Docker 自動輪轉 |
| InspectionReport 圖片 | 永久（直到手動刪除） | 無自動清理 |
| SensorReading | 180 天 | 排程清理（v1.2 規劃） |
| 備份 DB 快照 | 7 天 | 部署腳本自動清理 |
