## 1. CI/CD 整體架構

```
┌──────────────────────────────────────────────────────────────┐
│                     開發端（Dev Machine）                    │
│                                                              │
│  git push feature/* → PR → Code Review → Merge to develop  │
└───────────────────────────┬──────────────────────────────────┘
                            │ GitHub webhook
┌───────────────────────────▼──────────────────────────────────┐
│                   GitHub Actions（CI）                        │
│                                                              │
│  ┌─ PR Checks ─────────────────────────────────────────┐    │
│  │  lint → test → build-check → security-scan          │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │
│  ┌─ Release Build ─────────────────────────────────────┐    │
│  │  trigger: push tag v*.*.* or manual dispatch         │    │
│  │  build ARM64 → push to GHCR → create release        │    │
│  └──────────────────────────────────────────────────────┘    │
└───────────────────────────┬──────────────────────────────────┘
                            │
┌───────────────────────────▼──────────────────────────────────┐
│              GitHub Container Registry (GHCR)                │
│  ghcr.io/guessleej/xcloudvlmui-backend:v1.1.0               │
│  ghcr.io/guessleej/xcloudvlmui-frontend:v1.1.0              │
└───────────────────────────┬──────────────────────────────────┘
                            │ docker pull (on Jetson)
┌───────────────────────────▼──────────────────────────────────┐
│                   Jetson Orin（生產環境）                     │
│  install.sh → docker compose pull → docker compose up -d    │
└──────────────────────────────────────────────────────────────┘
```

## 2. GitHub Actions 工作流程

### 2.1 PR 檢查流程（`.github/workflows/ci.yml`）

```yaml
name: CI

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [develop]

jobs:
  backend-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install ruff black mypy
      - run: ruff check backend/
      - run: black --check backend/
      - run: mypy backend/ --ignore-missing-imports

  backend-test:
    runs-on: ubuntu-latest
    needs: backend-lint
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r backend/requirements.txt -r backend/requirements-dev.txt
      - run: pytest backend/tests/ --cov=backend --cov-report=xml
      - uses: codecov/codecov-action@v4
        with:
          files: coverage.xml

  frontend-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
          cache-dependency-path: frontend/package-lock.json
      - run: cd frontend && npm ci
      - run: cd frontend && npm run lint
      - run: cd frontend && npm run type-check

  frontend-test:
    runs-on: ubuntu-latest
    needs: frontend-lint
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
      - run: cd frontend && npm ci
      - run: cd frontend && npm test -- --coverage

  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Trivy vulnerability scan
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: "fs"
          scan-ref: "."
          severity: "CRITICAL,HIGH"
          exit-code: "1"
```

### 2.2 Release 建置流程（`.github/workflows/release.yml`）

```yaml
name: Release

on:
  push:
    tags:
      - "v*.*.*"
  workflow_dispatch:
    inputs:
      platform:
        description: "Target platform"
        required: true
        default: "linux/arm64"
        type: choice
        options: ["linux/arm64", "linux/amd64"]
      skip_vlm:
        description: "Skip vlm-webui"
        type: boolean
        default: true

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/guessleej/xcloudvlmui-backend
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=sha
      
      - name: Build and push backend
        uses: docker/build-push-action@v5
        with:
          context: backend/
          platforms: ${{ github.event.inputs.platform || 'linux/arm64' }}
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
      
      - name: Build and push frontend
        uses: docker/build-push-action@v5
        with:
          context: frontend/
          platforms: ${{ github.event.inputs.platform || 'linux/arm64' }}
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
      
      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        if: startsWith(github.ref, 'refs/tags/')
        with:
          files: |
            scripts/install.sh
            docker-compose.yml
          body_path: CHANGELOG.md
          draft: false
          prerelease: ${{ contains(github.ref, 'rc') || contains(github.ref, 'beta') }}
```

## 3. 多平台映像建置策略

### 3.1 平台矩陣

| 平台 | 目標裝置 | Make 指令 | 說明 |
|------|----------|-----------|------|
| `linux/arm64` | Jetson Orin（主要生產） | `make release` | 跳過 vlm-webui |
| `linux/arm64` | Jetson Orin（完整版） | `make release-full` | 含 vlm-webui |
| `linux/amd64` | x86_64 開發機 | `make release-amd64` | 本機測試用 |

### 3.2 Dockerfile 多階段建置（Backend）

```dockerfile
# backend/Dockerfile
# ── Stage 1: Builder ────────────────────────────────────────
FROM python:3.11-slim AS builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir --user -r requirements.txt

# ── Stage 2: Production ─────────────────────────────────────
FROM python:3.11-slim AS production

# 非 root 用戶
RUN useradd -m -u 1000 appuser

WORKDIR /app
COPY --from=builder /root/.local /home/appuser/.local
COPY . .

USER appuser
ENV PATH=/home/appuser/.local/bin:$PATH
ENV PYTHONPATH=/app

EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s \
  CMD curl -f http://localhost:8000/health || exit 1

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 4. build-release.sh 腳本設計

### 4.1 腳本用途

`scripts/build-release.sh` 為本機生產映像打包腳本，用於在 CI/CD 以外的情況（如無網路的部署環境）直接建置映像。

### 4.2 腳本使用

```bash
# 基本用法
bash scripts/build-release.sh [OPTIONS]

# 選項
--platform <platform>  目標平台（預設: linux/arm64）
--skip-vlm             跳過 vlm-webui 服務建置
--skip-mqtt            跳過 mosquitto 服務（使用外部 Broker）
--no-cache             不使用 Docker cache（全量重建）
--push                 建置後推送至 Registry
--tag <tag>            自訂映像 Tag（預設: 從 git describe）

# 範例
bash scripts/build-release.sh --platform linux/arm64 --skip-vlm
bash scripts/build-release.sh --platform linux/amd64 --no-cache
bash scripts/build-release.sh --platform linux/arm64 --push --tag v1.2.0
```

### 4.3 腳本流程

```
build-release.sh
├── 1. 解析命令列參數
├── 2. 檢查 Docker Buildx 是否可用
├── 3. 讀取 git tag 作為版本號（若未指定 --tag）
├── 4. 確認 .env 與 SSL 憑證存在
├── 5. 建置各服務映像
│   ├── backend
│   ├── frontend
│   ├── nginx（靜態設定）
│   ├── mosquitto（除非 --skip-mqtt）
│   └── vlm-webui（除非 --skip-vlm）
├── 6. 執行冒煙測試（啟動容器 → 呼叫 /health）
├── 7. 若 --push：推送至 Registry
└── 8. 輸出建置摘要（映像大小、耗時）
```

## 5. 部署流程

### 5.1 Jetson 首次部署

```bash
# 1. 下載 install.sh（從 GitHub Release）
curl -fsSL https://github.com/guessleej/xCloudVLMui/releases/latest/download/install.sh | bash

# install.sh 執行步驟：
# Step 1: 系統需求檢查（JetPack 版本、可用記憶體）
# Step 2: Docker 與 NVIDIA Container Toolkit 安裝確認
# Step 3: 下載 docker-compose.yml
# Step 4: 產生 SSL 憑證（gen-ssl.sh）
# Step 5: 複製 .env 範本，提示使用者填入設定
# Step 6: 從 GHCR 拉取映像（docker compose pull）
# Step 7: 下載 GGUF 模型（download-model.sh）
# Step 8: 啟動所有服務（docker compose up -d）
```

### 5.2 更新部署

```bash
# 拉取最新映像
docker compose pull

# 滾動重啟（zero-downtime for stateless services）
docker compose up -d --no-deps backend frontend nginx

# llama-cpp 需完整重啟（有狀態）
docker compose restart llama-cpp

# 確認健康狀態
make test
```

### 5.3 映像標籤策略

```
ghcr.io/guessleej/xcloudvlmui-backend:latest     → 最新穩定版
ghcr.io/guessleej/xcloudvlmui-backend:v1.1.0     → 特定版本
ghcr.io/guessleej/xcloudvlmui-backend:v1.1        → Minor 版本
ghcr.io/guessleej/xcloudvlmui-backend:sha-6e59416 → 特定 Commit
```

## 6. 品質閘門（Quality Gates）

### 6.1 PR 合併門檻

| 檢查項目 | 要求 | 失敗處理 |
|----------|------|----------|
| Backend Lint | ✅ 無警告 | 不可合併 |
| Backend Tests | ✅ 全通過 + 覆蓋率 ≥ 70% | 不可合併 |
| Frontend Lint | ✅ 無錯誤 | 不可合併 |
| Frontend Tests | ✅ 全通過 | 不可合併 |
| Security Scan | ✅ 無 CRITICAL/HIGH CVE | 需人工核准 |
| Code Review | ≥ 1 Approve | 不可合併 |

### 6.2 Release 門檻

| 檢查項目 | 要求 |
|----------|------|
| 所有 CI 檢查通過 | ✅ |
| 映像建置成功 | ✅ |
| 冒煙測試通過 | ✅ `/health` 回傳 200 |
| CHANGELOG 已更新 | ✅ |

### 6.3 冒煙測試端點

```bash
# scripts/test-services.sh 執行項目
```

## 7. 回滾策略

### 7.1 快速回滾（映像已在本機）

```bash
# 列出可用版本
docker images ghcr.io/guessleej/xcloudvlmui-backend

# 回滾至上一版
docker compose stop backend
docker tag ghcr.io/guessleej/xcloudvlmui-backend:v1.0.5 \
           ghcr.io/guessleej/xcloudvlmui-backend:latest
docker compose up -d backend

# 確認健康狀態
make test
```

### 7.2 資料庫回滾

```bash
# SQLite 資料庫備份還原
# （建議每次部署前自動備份）
cp data/xcloudvlm.db data/xcloudvlm.db.$(date +%Y%m%d_%H%M%S).bak

# 還原
cp data/xcloudvlm.db.20260410_143000.bak data/xcloudvlm.db
docker compose restart backend
```

### 7.3 回滾決策樹

```
發現問題
  │
  ├── 影響範圍：僅後端 API
  │     → docker compose restart backend（若為設定問題）
  │     → 或回滾 backend 映像
  │
  ├── 影響範圍：前端 UI
  │     → docker compose restart frontend
  │     → 或回滾 frontend 映像
  │
  ├── 影響範圍：LLM 推論
  │     → make restart-llm
  │     → 或回滾 llama.cpp 映像版本
  │
  └── 影響範圍：資料損毀
        → 停止所有服務
        → 還原 DB 備份
        → 重啟服務
        → 通知所有 Stakeholder
```
