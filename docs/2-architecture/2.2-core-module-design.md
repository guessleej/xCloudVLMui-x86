### 3.1 VHS Service

```python
class VHSService:
    """
    VHS 健康評分服務
    - 來源優先級：vlm > manual > seed > estimated
    - 14 天窗口滾動計算
    - DB 持久化儲存
    """
    
    SOURCE_PRIORITY = {"vlm": 4, "manual": 3, "seed": 2, "estimated": 1}
    TREND_DAYS = 14
    
    async def get_latest(self, equipment_id: int) -> VHSReading:
        """取得最新 VHS 評分，若無則回傳 estimated 預設值"""
        
    async def get_trend(self, equipment_id: int) -> list[VHSReading]:
        """取得 14 天趨勢資料"""
        
    async def write(self, equipment_id: int, score: float, source: str) -> VHSReading:
        """寫入 VHS 評分，自動更新 Equipment.vhs_score"""
        
    async def calculate_from_sensors(self, equipment_id: int) -> float:
        """從 MQTT 感測器資料計算 VHS（估算模式）"""
```

## 7. Middleware 設計

### 7.1 SyslogMiddleware

```python
class SyslogMiddleware(BaseHTTPMiddleware):
    """
    自動記錄所有 HTTP 請求至 syslog.db
    - 排除：/health, /docs, /openapi.json, /static
    - 記錄：method, path, status_code, duration_ms, timestamp
    - 錯誤（4xx/5xx）：額外記錄 response body 至 extra 欄位
    """
    
    EXCLUDE_PATHS = {"/health", "/docs", "/openapi.json", "/static", "/favicon.ico"}
    
    async def dispatch(self, request: Request, call_next) -> Response:
        if request.url.path in self.EXCLUDE_PATHS:
            return await call_next(request)
        
        start = time.time()
        response = await call_next(request)
        duration = (time.time() - start) * 1000
        
        level = "ERROR" if response.status_code >= 500 else \
                "WARNING" if response.status_code >= 400 else "INFO"
        
        await write_syslog(
            level=level,
            source="middleware",
            method=request.method,
            path=request.url.path,
            status_code=response.status_code,
            duration_ms=round(duration, 2)
        )
        return response
```

## 8. 設計決策紀錄

### ADR-001：雙資料庫設計

| 項目 | 決策 |
|------|------|
| 問題 | 事件日誌與主業務資料的隔離需求 |
| 決定 | 使用獨立 `syslog.db`，SyslogMiddleware 直接 SQLite 寫入 |
| 原因 | 避免高頻 log 操作影響主 DB 效能；syslog 不需要 ORM 關聯 |
| 風險 | 需維護兩個 DB 連線；備份策略需涵蓋兩個檔案 |

### ADR-002：VHS 來源優先級

| 項目 | 決策 |
|------|------|
| 問題 | 同一設備同一時間可能有多個 VHS 來源 |
| 決定 | `vlm > manual > seed > estimated` 優先級排序 |
| 原因 | VLM 推論結果最準確；人工輸入次之；種子資料兜底 |
| 風險 | 若 VLM 推論錯誤，需人工手動覆蓋（manual 模式） |

### ADR-003：Pipeline 並發執行

| 項目 | 決策 |
|------|------|
| 問題 | 四段式流水線各階段有部分可並發 |
| 決定 | 使用 `asyncio.gather()` 並發執行可獨立階段 |
| 原因 | 減少總推論時間；Jetson 多核心可充分利用 |
| 風險 | 並發錯誤調試難度較高；需為每個 gather task 設置 timeout |

### ADR-004：前端 Drawer 而非 Modal

| 項目 | 決策 |
|------|------|
| 問題 | 設備詳情展示空間需求 |
| 決定 | 右側滑入 Drawer，而非全屏 Modal |
| 原因 | 保留背景設備網格的視覺脈絡；多任務操作體驗更佳 |
| 風險 | 小螢幕（<768px）需額外處理 Drawer 全屏模式 |
