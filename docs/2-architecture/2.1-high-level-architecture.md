## 2. 資料流詳解

### 2.1 視覺巡檢流程

```
相機（USB/IP）
    │
    ▼ WebRTC
┌─────────────────┐
│  VLM WebUI      │  ← 使用者設定分析提示詞
│  live-vlm-webui │
│  :8090          │
└────────┬────────┘
         │ POST /v1/chat/completions（含 base64 影像）
         ▼
┌─────────────────┐
│  llama-cpp :8080 │
│  Gemma 4 E4B    │
└────────┬────────┘
         │ 診斷 JSON 結果
         ▼
┌─────────────────┐
│  FastAPI Backend │
│  /api/vlm/      │──→ VhsReading 寫入 SQLite
│  diagnose       │──→ Report 寫入 SQLite
└─────────────────┘
         │ API 回應
         ▼
    Next.js 儀表板
    （診斷結果、VHS 趨勢更新）
```

### 2.2 MQTT 感應器流程

```
工廠設備（各種感應器）
    │
    ▼ MQTT Publish  (xcloud/{device_id}/{sensor_type})
┌──────────────────────────────────────┐
│  Eclipse Mosquitto                   │
│  Broker :1883（原生）/ :9001（WS）   │
└────────────────┬─────────────────────┘
                 │ Subscribe xcloud/#
                 ▼
┌──────────────────────────────────────┐
│  Backend MQTT Listener               │
│  mqtt_service.py                     │
│  - Pydantic payload 驗證             │
│  - 閾值判定                          │
└─────┬────────────────────────────────┘
      │
      ├──→ MqttSensorReading 寫入 SQLite
      └──→ 如超出閾值: EquipmentAlert 寫入 SQLite
                 │
                 ▼
    Frontend 輪詢 /api/mqtt/readings/latest
    即時儀表板更新
```

### 2.3 RAG 知識庫查詢流程

```
使用者輸入問題
    │
    ▼ POST /api/rag/query
┌─────────────────┐
│  FastAPI Backend │
│  rag_service.py  │
└─────┬───────────┘
      │ POST /v1/embeddings（問題向量化）
      ▼
┌─────────────────┐      ┌─────────────────┐
│  llama-cpp      │      │  ChromaDB        │
│  /v1/embeddings │─────▶│  語意搜尋 top_k=5│
│  問題向量化      │◀─────│  相關文件片段回傳 │
└─────────────────┘      └─────────────────┘
      │ POST /v1/chat/completions（加入相關片段作為 context）
      ▼
┌─────────────────┐
│  llama-cpp      │
│  /v1/chat/      │
│  completions    │
└────────┬────────┘
         │ 回答 + 來源引用
         ▼
    Next.js RAG 頁面顯示
```

### 2.4 四段式巡檢管線狀態

```
GET /api/dashboard/pipeline-status
    │
    ▼ asyncio.gather（平行檢查）
    ├──▶ ① 視覺取像: HTTP GET vlm-webui :8090
    ├──▶ ② 邊緣推論: HTTP GET llama-cpp :8080/health + /v1/models
    ├──▶ ③ 知識整合: ChromaDB COUNT + rag_documents COUNT
    └──▶ ④ 維護輸出: equipment_alerts COUNT + reports COUNT
    │
    ▼ 彙等檢查結果
    PipelineStatusOut { stages[], overall, checked_at }
    │
    ▼
    Frontend 每 30s 輪詢更新 Pipeline Status Card
```

### 3.2 Port 映射表

| 外部 Port | 內部容器 | 對外開放 | 說明 |
|----------|---------|--------|------|
| 80 | nginx:80 | ✅ 對外 | 主要 Web 入口 |
| 1883 | mosquitto:1883 | ✅ 局域網 | MQTT 設備連線 |
| 8090 | vlm-webui:8090 | ⚠️ 內網 | WebRTC 視覺巡檢 |
| 8000 | backend:8000 | ⚠️ 內網 | API 直連（開發用）|
| 8080 | llama-cpp:8080 | ❌ 鎖定 | 推論引擎（僅內網）|
| 3000 | frontend:3000 | ❌ nginx 代理 | Frontend（由 nginx 代理）|
| 9001 | mosquitto:9001 | ⚠️ 內網 | MQTT WebSocket |
| 49152-49200/UDP | vlm-webui | ✅ 對外 | WebRTC ICE |

## 4. 健康檢查與啟動順序

### 4.1 服務依賴鏈

```
model-init ─────────────────────────────────────────┐
                                                     │ exit 0 後
llama-cpp (depends_on: model-init completed) ◄───────┘
    │ healthy (GET :8080/health = 200)
    ├──▶ backend (depends_on: llama-cpp healthy)
    │        │ healthy (GET :8000/api/health = 200)
    │        └──▶ frontend (depends_on: backend healthy)
    │                 │ healthy
    │                 └──▶ nginx（所有服務就緒）
    │
    └──▶ vlm-webui (depends_on: llama-cpp healthy)

mosquitto ── 獨立啟動，不影響其他服務
```

### 4.2 健康狀態判斷指標

```json
GET /api/health → 回傳：
{

  "llm_ok": true,
  "chroma_ok": true,
  "db_ok": true,
  "version": "1.1.0",
  "timestamp": "2026-04-10T..."
}
```

## 5. 安全邊界

### 5.1 網路隔離設計

```
外部網路
    │
    ▼ Port 80/443 (HTTPS + TLS 終端)
┌──────────────────────────────────────┐
│  Nginx                               │
│  - X-Frame-Options: DENY             │
│  - X-Content-Type-Options: nosniff   │
│  - Content-Security-Policy           │
└──────────────────────────────────────┘
              │
              ▼ Docker 內部網路
┌──────────────────────────────────────┐
│  內部服務（llama-cpp :8080 等）不對外 │
│  JWT 認證（NextAuth v5）              │
│  CORS: ALLOWED_ORIGINS 限制           │
└──────────────────────────────────────┘
```
