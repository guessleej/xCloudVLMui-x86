## 1. 專案概述（Project Overview）

**xCloudVLMui Platform** 是由 **云碩科技 xCloudinfo Corp.Limited** 開發，部署於 Advantech AIR-030
（Jetson AGX Orin 64GB）邊緣主機的**工廠設備健康管理平台**。整合視覺語言模型（VLM）、
檢索增強生成（RAG）與感測器物聯網（IoT），提供全自動設備健康評分、異常診斷、維保工單產生
與知識庫問答，取代傳統人工巡檢。

## 2. 業務目標（Business Objectives）

| # | 目標 | 量化指標 | 目標達成期 |
|---|------|---------|-----------|
| B1 | 減少設備非計畫性停機 | 停機次數減少 ≥ 30% | v1.2.0 |

## 3. 專案範疇（Project Scope）

### 3.1 範疇內（In-Scope）

- 邊緣端單機部署（Jetson AGX Orin AIR-030，離線環境）
- 設備 VHS 健康評分系統（0–100 分）
- VLM 視覺分析（Gemma 4 E4B GGUF，128K context）
- RAG 知識庫問答（ChromaDB + llama.cpp）
- MQTT 感測器資料採集與警報（Eclipse Mosquitto）
- Web 儀表板（Next.js 14，桌面瀏覽器）
- 維修報告產生（Markdown 格式，PDF 匯出待 v2.0.0）
- 系統日誌與 KPI 報告（syslog.db + kpi_report.py）
- CI/CD 自動化（GitHub Actions + GHCR）

### 3.2 範疇外（Out-of-Scope）

- 多廠房聯網監控（v2.0.0 功能）
- 雲端部署 / Kubernetes（v2.0.0 選項）
- Mobile App（不在計畫內）
- ERP / CMMS 系統整合（v2.0.0 功能）
- 機器學習模型訓練（使用預訓練 GGUF，不含 Fine-tuning）
- PLC 直接控制介面（監控 only，不含控制）

## 6. 約束條件（Constraints）

| 類型 | 約束 |
|------|------|
| 硬體 | Jetson AGX Orin 64GB RAM、ARM64、eMMC 64GB 儲存 |
| 網路 | 工廠內網，無外部網際網路連線（首次下載模型除外） |
| 電力 | 工業電源，允許 UPS 備援，容忍計畫性停電重啟 |
| 安全 | 設備資料不得離開廠區（不得上傳雲端） |
| 軟體授權 | 僅使用 Open Source 元件（Apache 2.0 / MIT / BSD） |
| 部署 | 單機 Docker Compose，不引入 K8s（v1.x） |
| 時程 | v1.1.0 已完成，v1.2.0 目標 2026-06-30 |

## 7. 假設（Assumptions）

| # | 假設 |
