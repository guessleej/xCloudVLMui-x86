## 2. 問題陳述與商業需求

### 2.1 問題陳述

製造業工廠設備（壓縮機、液壓系統、散熱風扇等）依賴人工定期巡檢，存在以下痛點：

- 🔴 **漏檢風險高**：巡檢周期長，設備異常難以及早發現
- 🔴 **知識孤島**：維修 SOP 分散於紙本或個人電腦，無法快速查詢
- 🔴 **故障診斷慢**：技術人員依賴經驗判斷，無法量化設備健康狀態
- 🔴 **資料孤立**：感應器資料未整合，無法與視覺檢測形成完整視野
- 🟡 **雲端依賴風險**：工廠網路隔離需求高，雲端 AI 服務不可行

### 2.2 商業需求

```
需求 1：邊緣端完全離線的 AI 視覺診斷能力
需求 2：感應器數據與 VLM 診斷的統一管理平台
需求 3：維修知識數位化與智慧檢索
需求 4：設備健康量化指標（VHS Score）與趨勢追蹤
需求 5：生產級容器化部署，一鍵安裝於 AIR-030
```

## 7. 假設與限制

### 7.1 假設

- AIR-030 硬體已完成 JetPack 6.0 安裝與 NVIDIA Container Toolkit 設定
- 首次部署時可連線 HuggingFace 下載 ~3.8GB 模型
- 工廠局域網已支援 MQTT Port 1883 通訊
- 使用者設備具備現代瀏覽器（Chrome 120+ / Edge 120+）

### 7.2 限制

- 模型推論速度受 Jetson AGX Orin GPU 限制（~30–50 tok/s）
- SQLite 不適合高並發（>100 同時連線），v2.0 評估升級 PostgreSQL
- WebRTC 需要 HTTPS 環境（自簽憑證或 Let's Encrypt）
- VLM WebUI 僅支援 ARM64 架構（JetPack 鏡像）
