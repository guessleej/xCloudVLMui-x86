01

模組概覽

Overview

MQTT 感測器管理頁面（/main/mqtt-devices）是 xCloud VLM Console 的設備後台管理中心。此頁面提供完整的設備生命週期管理功能，包含設備登錄、資訊編輯、歷史資料查看與警報閾值設定。

核心功能
設備 CRUD
新增、查看、編輯、刪除設備
歷史資料
趨勢圖

警報設定
四層閾值
危急下限/上限 + 警告下限/上限
佈局形式
Master-Detail

與監控頁面的定位差異

資料刷新每 5 秒自動刷新手動刷新（點選後更新）

歷史資料不提供提供趨勢圖（最近 80 筆）
閾值設定不提供提供四層閾值 CRUD

07

新增設備

Add Device

新增設備後，感測器即可透過 MQTT 發送資料並自動對應到此設備。

操作步驟

  1. 1
    點選右上角「＋ 新增設備」

    開啟新增設備對話框。

  2. 2
    填寫設備 ID（必填）

    用於 MQTT Topic 中的識別碼，全系統唯一。建議格式：英文小寫 + 底線 + 序號，例：compressor_01。

  3. 3
    填寫顯示名稱（必填）

    頁面顯示用的中文名稱，例：壓縮機 #1。

  4. 4
    設定設備類型、安裝位置（選填）

    設備類型選項：sensor / compressor / motor / pump / conveyor / cooler / panel / plc / other。

  5. 5
    填寫 Topic 前綴（必填）

    MQTT 主題前綴，格式為 xcloud/{device_id}。此前綴加上 /{sensor_type} 即為完整 Topic。

  6. 6
    點選「確認新增」完成

    設備新增後自動出現在左側清單。接著可至警報閾值區塊設定各感測器的告警條件。

欄位必填規則範例
設備 ID必填英文小寫、數字、底線；全系統唯一；建立後不可改compressor_01
顯示名稱必填任意字串壓縮機 #1
設備類型選填下拉選單選擇compressor
安裝位置選填任意字串廠房 A 區域 1
Topic 前綴必填格式 xcloud/{device_id}xcloud/compressor_01
備註選填任意文字說明主線壓縮機，型號 ABC-300
08

編輯設備資訊

Edit Device

可修改設備的顯示名稱、類型、安裝位置、Topic 前綴與備註（設備 ID 不可修改）。

操作步驟

  1. 1
    在左側清單點選要編輯的設備

    右側會顯示該設備的詳情面板。

  2. 2
    點選詳情面板右上角的「✏️ 編輯」按鈕

    開啟編輯對話框，欄位已預先填入目前的設備資訊。

  3. 3
    修改需要更新的欄位

    可修改：顯示名稱、設備類型、安裝位置、Topic 前綴、備註。

  4. 4
    點選「儲存」完成

    更新成功後對話框關閉，詳情面板自動刷新顯示最新資訊。

注意 設備 ID 建立後不可修改。若 Topic 前綴發生變更，須確認實際感測器的發送 Topic 也同步更新，否則新資料將無法對應到此設備。
09

刪除設備

Delete Device

可從左側清單刪除設備，刪除後該設備的所有歷史資料與閾值設定將一併移除。

  1. 1
    找到左側清單中要刪除的設備列
  2. 2
    點選該列右側的 🗑 垃圾桶圖示

    彈出確認對話框：「確定刪除『設備名稱』？所有感測資料將一併移除。」

  3. 3
    點選「確定」

    設備從清單中移除，若右側正在顯示該設備的詳情，右側會回到「選擇一台設備」的空白狀態。

警告：此操作無法復原 刪除設備時，以下資料將永久移除：① 設備基本資訊 ② 所有歷史感測讀值（mqtt_sensor_readings） ③ 所有警報閾值設定（mqtt_alert_thresholds）。請在刪除前確認資料已備份或不再需要。
10

警報閾值管理

Thresholds

在設備詳情面板的下方區塊，可為每種感測器類型設定四層警報閾值，設定完成後閾值會以參考線顯示在歷史趨勢圖上。

四層閾值結構

從低到高依序為四個層級，每台設備、每種感測器可獨立設定：

危急 警告 正常 警告 危急
↑ min_value（危急下限） ↑ warn_min（警告下限） warn_max（警告上限）↑ max_value（危急上限）↑
欄位名稱中文說明圖表線色
min_value危急下限低於此值為危急狀態，需立即處理紅色
warn_min警告下限低於此值進入警告區間，需關注橘色
warn_max警告上限高於此值進入警告區間，需關注橘色
max_value危急上限高於此值為危急狀態，需立即處理紅色

新增閾值步驟

  1. 1
    點選「＋ 新增閾值」按鈕

    位於警報閾值區塊右上角，點選後展開新增表單。

  2. 2
    選擇感測器類型（必填）

    從下拉選單選擇要設定的感測器類型，例：temperature、vibration、pressure 等。

  3. 3
    填寫單位（選填）

    例如：°C、bar、mm/s、rpm。此欄位僅作為顯示標示，不影響邏輯判斷。

  4. 4
    填寫四個閾值數值（依需求填寫）

    危急下限、警告下限、警告上限、危急上限。不填寫的欄位留空即可，系統不會設定該層閾值。

  5. 5
    設定啟用狀態（預設啟用）

    可選擇「啟用」或「停用」。停用的閾值不會顯示在圖表上。

  6. 6
    點選「儲存」

    閾值儲存後，圖表上立即出現對應的參考線，閾值列表也會新增此筆記錄。

刪除閾值

在已設定的閾值列表中，每列右側有 🗑 刪除圖示。點選後，該閾值設定與對應的圖表參考線會同時移除。

建議設定方式 以溫度感測器為例（設備正常運作溫度 40–70°C）：
危急下限（min_value）= 20°C　　警告下限（warn_min）= 35°C
警告上限（warn_max）= 75°C　　危急上限（max_value）= 90°C

12

與監控頁面的關係

Relation

管理頁面（本頁）與監控頁面（/main/mqtt）共用同一份資料庫，兩者互相連動：

在管理頁面做的事在監控頁面的效果
新增設備監控頁面的設備卡片列表新增對應卡片
修改設備顯示名稱監控頁面卡片標題更新為新名稱（下次刷新後）
刪除設備監控頁面卡片消失，相關歷史讀值移除
設定警報閾值（監控頁面目前不顯示閾值，僅在管理頁圖表呈現）

建議工作流程

  1. 1
    在管理頁面登錄新設備

    填寫設備 ID、名稱、類型、安裝位置、Topic 前綴。

  2. 2
    啟動實際感測器，確認資料傳入

    至監控頁面觀看設備卡片，確認感測數值已出現。或使用監控頁面的「測試發佈」功能驗證資料管道。

  3. 3
    回到管理頁面設定警報閾值

    點選設備 → 在警報閾值區塊設定各感測器的四層閾值。

  4. 4
    確認趨勢圖與閾值參考線正確顯示

    圖表上的閾值線符合設定值，代表整體設定完成。

常見問題排查

Troubleshoot
問題現象可能原因解決方式
新增設備後清單沒有更新 新增後頁面刷新有短暫延遲 點選右上角「↻ 刷新」按鈕手動更新清單
點選設備後右側一直轉圈 後端 API 載入詳情資料中，或網路異常 等待幾秒後點選「↻ 刷新」；若持續失敗確認後端容器是否正常運行
歷史圖表無資料顯示 該設備尚未接收到 MQTT 訊息 確認感測器已連線並發送資料；至監控頁面使用「測試發佈」驗證
閾值設定後圖表無參考線 目前查看的感測器 Tab 與閾值設定的類型不同 切換至閾值設定對應的感測器 Tab，圖表參考線即出現
設備類型下拉選單沒有想要的選項 系統預設類型不含此類型 選擇最接近的類型（如 other）即可，顯示名稱可自由設定
新增閾值後顯示「新增失敗」 同一設備的同一感測器類型已有閾值設定，或欄位格式錯誤 確認感測器類型未選擇重複；數值欄位只能填寫數字
設備顯示 Offline 感測器未發送資料，或 Topic 不符合 確認感測器 Topic 格式為 xcloud/{device_id}/{sensor_type} 且 device_id 完全符合
